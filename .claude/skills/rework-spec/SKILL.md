# /rework-spec — Spec Rework Workflow

You are the lead orchestrator for a spec rework pass. A spec PR has received review
comments from a human reviewer. Your job is to coordinate fixing those comments.

---

## Step 0 — Parse input and resolve paths

`$ARGUMENTS` is the PR number (e.g. `42`).

```bash
gh pr view $ARGUMENTS --json headRefName,title --jq '{branch: .headRefName, title: .title}'
```

The `headRefName` will be `spec/{FEATURE_NAME}`. Derive:

```
PR_NUMBER       = $ARGUMENTS
SPEC_BRANCH     = {headRefName from gh}
FEATURE_NAME    = {branch name with "spec/" prefix stripped}
SPEC_DIR        = .claude/tasks/{FEATURE_NAME}/
SPEC_PATH       = .claude/tasks/{FEATURE_NAME}/spec.md
TASKS_PATH      = .claude/tasks/{FEATURE_NAME}/tasks.json
```

---

## Step 1 — Fetch PR review comments

First, resolve the repository owner and name:
```bash
REPO=$(gh repo view --json nameWithOwner --jq '.nameWithOwner')
```

Then fetch all comment types:
```bash
# Get review comments (inline code comments)
gh api "repos/${REPO}/pulls/{PR_NUMBER}/comments" --jq '.[] | "**" + .user.login + "** on `" + .path + "` line " + (.line|tostring) + ":\n" + .body + "\n"'

# Get review body comments
gh api "repos/${REPO}/pulls/{PR_NUMBER}/reviews" --jq '.[] | select(.body != "") | "**" + .user.login + "** (review):\n" + .body + "\n"'

# Get issue-style comments
gh api "repos/${REPO}/issues/{PR_NUMBER}/comments" --jq '.[] | "**" + .user.login + "**:\n" + .body + "\n"'
```

Combine all comments into `ALL_COMMENTS`.

---

## Step 2 — Checkout spec branch

```bash
git checkout "spec/{FEATURE_NAME}"
git pull origin "spec/{FEATURE_NAME}"
```

---

## Step 3 — Spawn spec-writer agent to apply fixes

1. Read `.claude/skills/create-spec/phase-write.md`
2. Spawn the **spec-writer** agent with that content, appending:

```
--- CONTEXT ---
FEATURE_NAME:  {FEATURE_NAME}
FEATURE_DESC:  (rework pass — see review comments below)
SPEC_DIR:      {SPEC_DIR}
SPEC_PATH:     {SPEC_PATH}
TASKS_PATH:    {TASKS_PATH}

--- REVIEW COMMENTS ---
{ALL_COMMENTS}

IMPORTANT: This is a rework pass. The spec and tasks files already exist.
Read the existing files first, then apply the changes requested in the review comments.
Do NOT rewrite the entire spec — only modify what the comments ask for.
```

3. Wait for the agent to finish.

---

## Step 4 — Commit and push

```bash
git add ".claude/tasks/{FEATURE_NAME}/"
git diff --cached --quiet || git commit -m "spec({FEATURE_NAME}): address PR review comments"
git push origin "spec/{FEATURE_NAME}"
```

---

## Step 5 — Reply to PR

```bash
gh pr comment {PR_NUMBER} --body "$(cat <<'EOF'
## Spec rework complete

Addressed review comments and pushed updates to `spec/{FEATURE_NAME}`.

**Changes:**
{bullet list of comments addressed}

Generated by the /rework-spec workflow.
EOF
)"
```

---

## Step 6 — Return to main and report

```bash
git checkout main
```

Report the PR URL and summary of changes.

---

## Global rules

- Only address comments from the PR — do not gold-plate
- If a comment is ambiguous, implement the most conservative interpretation
- The spec-writer agent works interactively for ambiguous comments
- Keep a running status log
