## Codebase Patterns
- Client models live in `libs/chairly/src/lib/clients/models/` — never in `util/`
- API service follows StaffApiService pattern: inject HttpClient + API_BASE_URL, one method per HTTP verb
- Spec files use HttpTestingController from `@angular/common/http/testing`
- models/index.ts must use `export type { ... }` (not `export { ... }`) for interface re-exports — required by isolatedModules
- sheriff.config.ts chairly-lib depRules must include each new domain (e.g. 'domain:clients') when adding a new domain
- Run `npx nx run chairly-lib:lint --fix` to auto-fix simple-import-sort errors quickly
- `viewChild.required<ComponentType>('templateRefName')` for typed access to template ref components
