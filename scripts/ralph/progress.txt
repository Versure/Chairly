## Codebase Patterns
- Mediator namespace: `Chairly.Api.Shared.Mediator` (matches folder structure). Requires: CA1716/CA1724 suppressed via .editorconfig for Shared/Mediator files; MA0049 suppressed via pragma in Mediator.cs (class name matches namespace segment)
- All types in Chairly.Api must be `internal` (CA1515 is an error, not a warning)
- CA1812 fires for DI-registered internal classes — suppress with `#pragma warning disable CA1812`
- Exception types need standard constructors (CA1032/RCS1194) — use pragma suppression if intentional
- LINQ `GroupBy`/`ToDictionary` must pass `StringComparer.Ordinal` (MA0002 is an error)
- All `await` calls need `.ConfigureAwait(false)` (MA0004/CA2007 are errors)
- Delegate type name ending in "Delegate" needs `#pragma warning disable CA1711`
- Each type must be in its own file matching the type name (MA0048)
- IPipelineBehavior parameter `next` must be renamed (CA1716 — VB keyword) — use `continuation`
- Test files in `Chairly.Tests/Shared/Mediator/` need `#pragma warning disable CA1716` + `using MediatorImpl = Chairly.Api.Shared.Mediator.Mediator;` alias (namespace ending in 'Mediator' causes CS0118 ambiguity)
- For test-only DI-registered classes: use `services.AddSingleton<IInterface>(new ConcreteClass())` instead of type-based registration to avoid CA1812
- OpenAPI in .NET 10: requires both `Microsoft.AspNetCore.OpenApi` (for AddOpenApi/MapOpenApi) AND `Scalar.AspNetCore` (for MapScalarApiReference) — `AddOpenApi()` is NOT built into the SDK, it's a separate package
