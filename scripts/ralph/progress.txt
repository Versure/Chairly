## Codebase Patterns
- API services use `inject(API_BASE_URL)` token from `@org/shared-lib` for base URL (not hardcoded `/api`)
- `simple-import-sort` ESLint plugin enforces export sort order — run `nx run chairly-lib:lint --fix` to auto-fix
- Models use `export type` (not `export`) in index.ts re-exports
- Staff API: deactivate/reactivate use PATCH with `null` body
- `[style.xxx]` property bindings require `allowBindToStyle: true` in ESLint `no-inline-styles` rule — already configured
- Output names `save`/`cancel` are native DOM events — use `saved`/`cancelled` to satisfy `@angular-eslint/no-output-native`
- Empty `<button>` elements (icon-only, color circles) need `[attr.aria-label]` to satisfy `@angular-eslint/template/elements-content`
- Color group labels: use `<p>` instead of `<label>` when no single form control can be associated
- Test selector tip: when a component has multiple `button[type="button"]` elements, find by textContent: `buttons.find(b => b.textContent?.trim() === 'Annuleren')`
- Two-way bindings: use `model<T>()` (ModelSignal) instead of `input()` + `output()` — `@angular-eslint/prefer-signal-model` enforces this; `model.set()` both updates locally and emits `${name}Change` to parent
- `TestBed.flushEffects()` is deprecated in Angular 21 — avoid `effect()` for sync patterns; prefer `model()` where possible
- When adding a new domain, update `sheriff.config.ts` `chairly-lib` dep rules to include `domain:<new>`
- Components with subscriptions need Subject+takeUntil: `private destroy$ = new Subject<void>(); ngOnDestroy() { destroy$.next(); destroy$.complete(); }` with `.pipe(takeUntil(this.destroy$))`
- `vitest/no-conditional-expect`: never put `expect()` inside `if` blocks — assert outside, then use `x!.method()` if needed
