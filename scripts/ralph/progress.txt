## Codebase Patterns
- EF Core migrations auto-generate with BOM/encoding issues — always run `dotnet format` before `--verify-no-changes` to auto-fix them
- `Conflict` type is NOT in the OneOf library — create a custom `internal readonly struct Conflict` in `Chairly.Api/Shared/Results/` and add `#pragma warning disable CA1716` (VB keyword conflict on "Shared")
- The `.editorconfig` suppresses CA1716 only for `Chairly.Api/Shared/Mediator/**` — new Shared subfolders need their own pragma or editorconfig entry
- `InternalsVisibleTo("Chairly.Tests")` is set in `Chairly.Api.csproj`, so internal types in the API project are accessible from tests
- CA1515 fires on `public` types in the API project — always use `internal` for handler classes and result types
- GetClientsListHandler references CreateClientHandler.ToResponse — these two handlers are coupled and must be committed together
