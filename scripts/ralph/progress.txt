## Codebase Patterns
- Migration output-dir is `Migrations` (relative to --project), NOT `Persistence/Migrations`. Existing migrations are in `src/backend/Chairly.Infrastructure/Migrations/`.
- EF Core-generated migration files need `dotnet format` run after creation (encoding fix).
- Domain entities with collection navigation properties: use `List<T>` with `#pragma warning disable CA2227, CA1002, MA0016` for EF Core OwnsMany.
- Configuration methods exceeding 60 lines: split into private static helper methods (MA0051 rule).
- DesignTimeDbContextFactory is in `Chairly.Infrastructure/Persistence/` with `#pragma warning disable CA1812`.
- MA0132: Never implicitly convert `DateTime` to `DateTimeOffset` — use `new DateTimeOffset(dt, TimeSpan.Zero)` explicitly.
- `BookingMapper.ToResponse` is the shared mapper for all booking handlers — lives in `Chairly.Api/Features/Bookings/BookingMapper.cs` (context root, not inside a slice).
- Date filter in EF Core: use explicit `DateTimeOffset` range (`>= startOfDay && < endOfDay`) rather than `.Date` comparison (always translatable).
- MA0002: `new Dictionary<string, string[]>` triggers MA0002 — always pass `StringComparer.Ordinal` as constructor argument.
- MA0051: Handler methods exceeding 60 lines must be split into private helpers (e.g., `HasOverlapAsync`, `CreateBooking`).
- Command classes with `List<T>` properties need `#pragma warning disable CA2227, CA1002, MA0016` on the property for model binding.
